# Synology 安全盤點與整合藍圖 v1

## 一、目前已確認的事實

### 裝置與入口
- NAS 類型：**Synology**
- 網域：**site2000.com**
- DSM 管理入口：**https://www.site2000.com:5001**

### 憑證狀態
- 原本錯誤原因：DSM 對外使用 `synology` 預設憑證
- 已修正：成功申請 **Let's Encrypt** 憑證給 `www.site2000.com`
- 驗證結果：
  - CN = `www.site2000.com`
  - SAN = `site2000.com`, `www.site2000.com`

---

## 二、外部暴露面初步盤點

### 對外可達 Port
- **80**：OPEN
- **443**：OPEN（但目前 HTTPS 主站行為異常 / reset）
- **5000**：OPEN
- **5001**：OPEN
- **22**：CLOSED

### 初步觀察
1. **5001** 現在是 DSM 管理入口，且憑證已修正
2. **5000** 仍對外開放，代表 HTTP 管理入口仍暴露
3. **80** 對外提供一個 nginx 網站
4. **443** 有開，但目前 `https://www.site2000.com` 連線異常（reset）

---

## 三、安全判斷

### 正面項
- DSM 已有 MFA / email 驗證
- SSH 22 沒有直接暴露
- 5001 憑證問題已修正

### 風險項
1. **DSM 5000/5001 直接暴露在公網**
   - 不算立即錯誤，但屬高敏感入口
2. **5000 還開著**
   - HTTP 管理入口不應長期直接暴露
3. **443 異常**
   - 代表主站 HTTPS 目前配置不完整或有反向代理 / TLS 設定問題
4. **管理入口仍用主網域 + port**
   - 長期來看不夠優雅，也不利於分層

---

## 四、我建議的安全改善順序

### 第一優先：收斂 DSM 暴露面
1. 關閉或限制 **5000** 對外存取
2. 盡量不要讓 DSM 直接長期裸露在公網
3. 若需要遠端管理，優先考慮：
   - Tailscale
   - VPN
   - 受限來源 IP
   - 反向代理 + 額外保護

### 第二優先：處理 443 主站異常
1. 釐清 `https://www.site2000.com` 為什麼 reset
2. 確認是否有：
   - 反向代理配置錯誤
   - TLS 版本/套件問題
   - nginx upstream 配置問題
   - 憑證綁定錯誤

### 第三優先：把管理入口和公開網站分層
建議未來改成：
- `www.site2000.com` → 公開網站
- `nas.site2000.com` 或 `admin.site2000.com` → NAS / DSM 管理入口

這樣比較乾淨，也比較容易治理。

## 五、Synology 在整體架構中的建議角色

我建議 Synology 不只是備份機，而是：

### 角色 1：私有主權記憶庫
適合放：
- 敏感資料
- 長期保存文件
- 封存資料
- 最終版本文件

### 角色 2：冷資料與備份層
適合放：
- 不常動的大檔
- 歷史專案
- 歷史帳務 / 合約 / 報價
- Dropbox / Google Drive 的備份鏡像

### 角色 3：可信設備才可開啟的內部節點
這點很符合 Jeff 的思路：
- 雲端容器是可索引的記憶層
- NAS 是私有、可信、主權化的核心記憶層
- 終端設備是開啟記憶層的 key

## 六、整體整合藍圖（目前建議）

### 雲端記憶層
- Dropbox
- Google Drive

用途：
- metadata-first
- on-demand import
- 外部協作與快速檢索

### 私有主權層
- Synology NAS

用途：
- 長期保存
- 敏感資料
- 備份 / 封存
- 最終版本保存

### 入口與身份層
- `site2000.com`

用途：
- 入口頁
- 後續可做反向代理與服務分流
- 未來可區分 public vs admin vs app

### 助理執行層
- OpenClaw workspace + Joe

用途：
- 查詢
- 索引
- 帶檔
- 工作區處理
- 歸位 / 回寫規劃

## 結論

Jeff 的整合方向是成立的：
- 雲端記憶容器（Dropbox / Google Drive）
- 私有主權記憶層（Synology NAS）
- 網域入口（site2000.com）
- 助理執行層（Joe / workspace）

目前最迫切的事不是再加功能，而是：

> **先把 Synology 的對外暴露與入口邏輯收乾淨。**
